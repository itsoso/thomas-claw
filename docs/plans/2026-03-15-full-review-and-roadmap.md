## 核心诊断

### 问题1: 私信系统不工作（阻塞性）
- 4条私信发出，0条回复检测到
- DM面板打开不稳定（DOM选择器脆弱）
- 没有持续跟进机制：发了就忘
- **影响**: 整个转化链断在这里

### 问题2: 直播间选择不精准
- 去了书法/钢琴/唱歌直播间 = 浪费时间
- 大直播间(1000+人)弹幕淹没 = 没人看到你
- **应该**: 只去5-50人的聊天类小直播间

### 问题3: AI回复质量
- 连续夸奖同一个话题（"好听""真棒"×6）
- 没有利用直播间上下文创造记忆点
- 主播跟我说话时没有精准接住

### 问题4: 反检测不足
- 单IP单账号高频操作
- 弹幕频率有时过高
- 没有休息间隔

## 优化路线图

### Phase A: 修私信（最高优先 ⚡）
1. 重写 DM 面板检测（用 Playwright locator 而非 evaluate）
2. 添加 DM 失败截图用于调试
3. 启动时检查所有已私信主播的回复
4. 对话阶段化：破冰→了解→提议加微信
5. 添加 DM 重试：失败后下次来访再试

### Phase B: 深耕转化
1. 高潜力主播池：从15个有反馈的里选3-5个
2. 每天回访，停留更长(20-40分钟)
3. AI记忆连续性："上次你说的XX，今天怎么样了"
4. 只在高潜力主播直播间送大礼(99-520钻)

### Phase C: 精准发现
1. 只找5-50人在线的小直播间
2. 优先新人主播（标题含"第一天""新人"）
3. 进入后检查：主播在聊天还是表演？不聊天就走
4. 语音检测：90%是音乐/唱歌 → 2分钟内离开

### Phase D: 反检测加固
1. 每个直播间限弹幕5条/次
2. 直播间间隔5-15分钟随机
3. 每天运行不超过4小时
4. 弹幕内容微调（随机空格/符号）

### Phase E: 看板和数据
1. ✅ Web Dashboard（已完成，需修Tab）
2. 看板持久化到文件（已完成）
3. 转化漏斗可视化
4. 每日报告自动生成
